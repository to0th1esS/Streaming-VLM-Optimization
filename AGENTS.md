# 项目协作约定

## 代码注释

- 后续新增或修改代码时，对算法依据、边界条件、状态更新和非显然实现添加简洁中文注释。
- 注释应解释“为什么这样设计”或“该逻辑保证什么”，避免逐行复述代码。
- 公共参数、实验策略和回退兼容逻辑需要注明中文含义。
- 英文专业术语首次出现在注释中时，同时附上中文解释。
- 不为简单赋值、直接函数调用等自解释代码添加冗余注释。

## 实验记录

- 所有研究实验继续按时间顺序追加到 `docs/research_iteration_log_zh.md`。
- 记录实验目的、代码提交、数据配置、模型配置、结果路径、核心指标、现象解释和下一步判断。
- 关键研究节点建立 Git 标签，保证可以回退。
